1. Instructions

  Run your server in the same computer where your SMART SEM server is installed. Make sure that your SMART SEM application is running and that you have 
your sample completely set up in the FIB/SEM before starting your server.  

Select your ip (ip´s are obtained from the system automatically) and your port. The default port is 8095 but could be changed to any other port. Be aware that if you change 
your IP or your Port you will need to stop and start again. If you want to use the server in a localhost select the 127.0.0.1 address or tick the box Use localhost.

Push START to start the server.Before closing your aplication, stop the server using the STOP button.

2. MSite CServer API

To find more about server code design, click here.

The server opens a TCP/IP socket listener in the specified address and port. From the GUI a socket connection must be opened, a message can be sent in form of a string.
Here an example for grabbing a frame from the SEM microscope screen is shown using python with the socket library.

import socket

path = "C:\Test\\trialsGrab"

payload = { 'function' : 'setDirFrames' , 'dir' : path }  % set path for saving grab

client_socket = socket . socket ( socket . AF_INET , socket . SOCK_STREAM ) % open socket

client_socket . connect (( '127.0.0.1' , int ( 8095 )))

client_socket . sendall ( str ( payload )) % send payload as string

answer = client_socket . recv ( 1024 )

 

% Now the directory is set up and we proceed to grab our frame

payload = { 'function' : 'grabFrame' , 'frame_name' : 'Image' }

client_socket = socket . socket ( socket . AF_INET , socket . SOCK_STREAM )

client_socket . connect (( '127.0.0.1' , int ( 8095 )))

client_socket . sendall ( str ( payload ))

answer = client_socket . recv ( 1024 )

 

Errors are the same as is expressed in the Zeiss API Manual, except for the NO_ERROR value, which means the execution of the function worked properly.

1.       getStagePositionXY

Gets coordinates X and Y from the microscope.

{ 'function': 'getStagePositionXY'}

 

Default RESPONSE 

{'xpos': '0.0',   ‘ypos': '0.0', ‘error’:’NO_ERROR’}

 

Similar methods:

·          getStagePositionXYZ 

·          getStagePositionComplete  (acquires x,y,z,r,t,m)

 

 

2.       setStagePositionXY

Sets coordinates X and Y in the microscope.

ATTENTION: Pay attention to the SCALE, ALL VALUES FROM THE DATA STAGE MUST BE SENT IN METERS!!!!

{ 'function': 'setStagePositionXY','xpos':0.0740 , 'ypos': 0.0850}

 

Default RESPONSE 

{ ‘error’:’NO_ERROR’}

 

Similar methods:

·          setStagePositionXYZ 

·          setStagePositionComplete  (sets x,y,z,r,t,m)

 

3.       setDirFrames

Sets the directory (accessible from the microscope computer) where the images grabbed are going to be stored.

{ 'function' : 'setDirFrames' , 'dir' : path }

 

RESPONSE 

{ ‘error’:’NO_ERROR’}

 

4.       grabFrame

Grabs a frame and puts the name to the image specified by the call. By default it also adds the time of acquisition.  Pictures are saved as .tif with a metadata header.

{ 'function' :'grabFrame', 'frame_name':'Image'}

RESPONSE 

{ ‘error’:’NO_ERROR’}
